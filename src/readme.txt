Новостной портал для туризма, где зарегистрированные пользователи смогут обмениваться новостями в сфере туризма.

Step 1.

Пока реализован раздел новости, а именно: CRUD операции.
Использован Spring Boot
База данных PostgreSQL
для ее создания использованы changelog's Liquibase
Также реализованы дополнительные фишки в плане отображения количества просмотров и дата добавления записи
Для отображения view использован Thymeleaf и Bootstrap

Step 2.

Пофиксил метод подсчета просмотров. Предыдущий работал некорректно.
Строка с датой просмотра изменена на интерактивную(при изменении записи "Добавлено: " меняется на "Изменено:"

Добавлены пользователи и роли. Реализован login и logout с помощью Spring Security.
Добавлена защита на уровне представлений. А именно: на определенные эндпоинты пускаем только авторизованных пользователей.
либо пользователей с определенными правами
Реализована форма регистрации
Добавлен Lombok

Step 3.
Изменен метод перехода на страницу пользователя. Теперь переход по логину.
Добавлен переход авторизованного пользователя на свою страницу из хедера в любом эндпоинте.

Добавлены пагинация и сортировка по дате на странице "Новости"
Оформлена главная страница с правилами сайта.
Добавлена публикация последней новости на главной странице путем обратной сортировки по дате и поиском первой новости.
Добавлен столбец "person_id" в таблицу "news". У каждой статьи теперь будет автор.
Доработана секьюрность на уровне представления Thymeleaf:
1) изменять и удалять новость могут только "superadmin", "admin" и пользователь, опубликовавший новость;
2) изменять и удалять карточку пользователя могут только "superadmin", "admin" и сам пользователь-обладатель этой карточки.
Изменена таблица "news", добавлено количество знаков для ввода в столбец "full_text".

Step 4.
Добавлена валидация при регистрации пользователя и публикации новости:
1) подключена зависимость spring-boot-starter-validation и использованы стандартные аннотации
пакета javax.validation.constraints @NotBlank, @NotNull, @Email;
2) написана собственная аннотация @Login для проверки поля логин на незанятость при регистрации;
(если логин занят пользователю выводится сообщение о данном событии).
3) написана собственная аннотация @Role для проверки валидности ввода в поле "Роль" администрацией сайта;
4) с помощью аннотаций @NotBlank и @Size установлены минимальный и максимальный порог символов
для полей "Заголовок", "Анонс" и "Текст".

Настроена должным образом админ-панель:
1) написан AdminController пользователи и администрация разведены по разным эндпоинтам
2) написан UpdateByPersonDto для изменения своих данных самим пользователем
3) добавлены новые views для админ-панели

Использовал @Builder заменяющий одноименный паттерн. Сократил количество строк кода.
Настроено шифрование с помощью BCrypt. Пароли теперь зашифрованы и не хранятся в чистом виде в бд.

Step 5.
Добавлена возможность оставлять комменты для каждой новости.
Реализован CommentsController.
Оставлять коммент может только зарегистрированный пользователь.
У каждого коммента есть автор, дата добавления и принадлежит конкретной новости.
Валидация ввода при добавлении.
Доработал метод сортировки комментариев по дате, добавлена пагинация.
